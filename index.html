Accessibility Testing

What is the p5 widget?

This is a reusable widget made by Atul Varma that makes it easier to embed editable p5.js sketches in blog posts, interactive curricula, and other places.

Test 1 - The Navbar

The p5 widget frame contains a code editor, an ouput frame and a toolbar. In the widget below, we will take a look at the toolbar and it's contents.

Test 2 - The Play Button

Go ahead and press the Play button. This will run the code that you have typed. It will also create a Stop Button right next to the play button

Test 3 - The Coding Area

After the nav bar, you should find the main coding area. Type the following code and you will be able to read and execute it: function setup() { createCanvas(200,200); background(255); }

Test 4 - Changing the code

In the code, the line which reads - 'createCanvas(200,200)' is the line that defines the size of the output. Go ahead and change the line to 'createCanvas(400,400)'. Now when you go to the nav bar, you will notice that there are 2 new buttons now: "Undo" and "Revert". These will help with changing the code back to previous versions.

Test 5 - Text Output

In the next widget, the code draws a ball on the output. Go into the nav bar, select "Text Output" and then press "Play." This will take you to the frame containing the text output. Enter the frame and go through the output. Are you able to find the description of the ball? Feel free to modify the code and press play again. Can you notice how the text output changes?

Test 6 - Grid Output

In the next widget, the code draws a ball on the output. Go into the nav bar, select "Grid Output" and then press "Play." This will take you to the frame containing the accessible output. Enter the frame and go through the output. Are you able to find the description of the ball?

Test 6 - More on Grid Output

In the next widget, the code draws a moving ball on the output. Go into the nav bar, select "grid output" and then press "Play." This will take you to the frame containing the accessible output. Enter the frame and go through the output. Are you able to find the description and direction of movement of the ball?

Test 7 - Sound Output

Please wear earphones/headphones for this test. In the next widget, the code draws a moving ball on the output. Go into the nav bar, select "sound output" and then press "Play." You should be able to hear the sound output. The mapping is as follows - the sounds move from left to right earphone as the object moves from left to right. The frequency reduces as the object moves from top to bottom.

Test 8 - More on Sound Output

Please wear earphones/headphones for this test. In the next widget, the code draws 2 moving balls on the output. Go into the nav bar, select "sound output" and then press "Play." You should be able to hear the sound output. The mapping is as follows - the sounds move from left to right earphone as the object moves from left to right. The frequency reduces as the object moves from top to bottom.
